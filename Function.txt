# Coordinate Transformation Notes

This file is documentation only. The authoritative implementation lives in
`universal-coordinate-converter.html`.

Do not copy formulas from this document back into the application. When formula
logic changes, update the HTML implementation first, run `python
tests/run_validation.py`, then update this document with a short description of
the change.

## Source of Truth

- Main implementation: `universal-coordinate-converter.html`
- Regression tests: `tests/run_validation.py`
- Validation notes: `VALIDATION.md`

## Supported Transformations

### Gauß-Krüger (Bessel, Potsdam) to WGS84

Live function names:

- `gk2geo(rw, hw)`
- `pot2wgs(lng, lat)`

Flow:

1. Convert Gauß-Krüger easting/northing to Bessel/Potsdam geographic coordinates.
2. Apply the live Potsdam-to-WGS84 datum shift.
3. Return WGS84 latitude and longitude.

Notes:

- The live HTML uses a seven-parameter-style `pot2wgs` transform with translation,
  rotation, and scale terms.
- Regression baseline in the current test suite:
  - Input: `3568189.267 5657692.868`
  - Output: latitude `51.0503134303347`, longitude `9.971401877600515`

### WGS84 to Gauß-Krüger (Bessel, Potsdam)

Live function names:

- `wgs2pot(lng, lat)`
- `Dezimal2GK(lng, lat)`

Flow:

1. Shift WGS84 geographic coordinates to Potsdam/Bessel coordinates.
2. Project the shifted coordinate to Gauß-Krüger easting/northing.

Notes:

- The current reverse datum shift is intentionally preserved because the app is
  already tuned around this behavior.
- The app limits GK output to the Germany-focused range used by the live
  `Dezimal2GK` function.

### SWEREF99 18 00 to WGS84

Live function name:

- `sweref99ToWGS84(easting, northing)`

Flow:

1. Remove SWEREF99 18 00 false easting/northing.
2. Apply the inverse Transverse Mercator calculation on the GRS80 ellipsoid.
3. Return WGS84 latitude and longitude.

Current projection constants:

- Ellipsoid: GRS80
- Semi-major axis: `6378137.0`
- Flattening: `1 / 298.257222101`
- Latitude of natural origin: `0.0`
- Central meridian: `18.0`
- Scale factor: `1.0`
- False easting: `150000.0`
- False northing: `0.0`

### WGS84 to SWEREF99 18 00

Live function name:

- `wgs84ToSweref99(lat, lng)`

Flow:

1. Convert WGS84 latitude/longitude to radians.
2. Apply the live forward Transverse Mercator calculation on the GRS80 ellipsoid.
3. Return SWEREF99 18 00 easting and northing.

Regression baseline in the current test suite:

- Input: latitude `55.12345678`, longitude `18.98765432`
- Output: easting `213008.7865462337`, northing `6111419.641371732`

## Historical Drift Resolved

Before 2026-05-20, this file contained executable-looking JavaScript that did
not fully match the application:

- `pot2wgs` and `wgs2pot` used different datum shift parameters than the live
  HTML implementation.
- `wgs84ToSweref99` used a different forward projection algorithm than the live
  HTML implementation.

That duplicate code was removed so maintainers do not accidentally use outdated
formulas. The HTML file is now the only implementation source.

## Validation Policy

- Do not change transformation formulas without adding or updating validation
  cases.
- Preserve the context that the project owner has manually checked the current
  converter output against actual field locations over several months of real
  use, with matching practical results.
- Run `python tests/run_validation.py` after every functional update.
- After each functional update, commit and push the updated project to GitHub.
